The Context

A high-volume Food & Beverage operation in Yorkshire was struggling with inbound data integrity. The ERP was receiving "dirty data" daily, leading to incorrect Best Before dates (BBE), stock-count discrepancies, and failed audit trails.

The Problem: Heroics vs. Process

The original Goods-In process was interpretative. Operators had to manually decide if a batch was correct, manually calculate shelf-life, and manually enter data into the WMS.

  • ✕

    High Cognitive Load: Operators were making complex decisions under pressure, leading to a 30-50% error rate.

  • ✕

    Silent Costs: Errors were only caught weeks later during production or audits, requiring expensive retrospective correction.

The Solution: Deterministic Layer

I implemented a bespoke VBA/SQL integration that acts as a hard firewall. Instead of training people to "be more careful," we changed the environment so that it is physically impossible to submit invalid data.

Key Features:

  • Real-time SQL validation against ERP purchase orders.
  • Automatic BBE and Shelf-Life logic enforcement.
  • Batch uniqueness checks at the point of scanning.
  • Simplified UI focusing only on essential data signals.
